# MovieThrow website

A single self-contained landing page (`index.html` + the app icon). No build step, no dependencies.

## Preview locally
```bash
open index.html
```

## The download link
The "Download" buttons point at **`MovieThrow.dmg`** (relative). Pick one:
- **Host the DMG alongside the page** — copy `build/release/MovieThrow.dmg` into this folder when you
  deploy. **Don't commit it to git** (it's ~57 MB; it's already covered by the repo `.gitignore`).
- **Or link to a release** — replace `href="MovieThrow.dmg"` (two spots in `index.html`) with your
  GitHub Releases URL, e.g. `https://github.com/<you>/<repo>/releases/latest/download/MovieThrow.dmg`.

## Deploy
- **Netlify / Cloudflare Pages / Vercel** — point the project at this `website/` folder (or drag it
  into Netlify Drop). Easiest if you also drop the DMG in here.
- **GitHub Pages** — Pages "deploy from branch" only serves the repo root or `/docs`, so either rename
  this folder to `docs/` or use a Pages Action. Upload the DMG as a Release asset and use that URL.
- **Any static host / S3** — upload `index.html`, `icon.png`, `icon-256.png` (and the DMG if hosting
  it here).

## Tweaking
Brand colors live in the `:root` CSS variables (`--blue`, `--violet`). The icon is generated by
`../make_appicon.swift`; re-run it and re-copy `icon_512.png` → `icon.png` if you change the design.
